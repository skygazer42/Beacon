# 系统管理

系统管理包含系统配置、账号权限、授权管理、API 安全、开发入口和告警声管理；平台运维分组另提供平台概览、诊断、审计、升级与设备扫描。

## 系统设置

**系统管理 → 系统设置** 显示当前配置摘要和 JSON 编辑区，并提供：

- 保存系统配置；
- 导出完整配置；
- 导入前预览，再按合并策略导入；
- 查看配置快照并回滚；
- 导出基础日志包或包含流媒体日志的诊断包。

保存会写入运行配置，但并非所有字段都能无中断热更新。端口、推理后端、密钥和进程启动参数变化后，应按部署文档重启相关服务并重新验收。

配置导出可能包含部署信息。对外分享前必须检查并移除地址、账号、Token、摄像头凭据和客户标识。

## 账号权限

**系统管理 → 账号权限** 使用 Django 用户模型和 Beacon 功能权限：

- 新建用户，设置初始密码、邮箱、姓名、启用状态、管理员和超级用户标记；
- 编辑用户或重置密码；
- 启用、停用、单个删除或批量删除用户；
- 为普通用户逐项勾选后端返回的功能模块权限。

当前实现没有可复用的 `Role`、资源域或租户角色编辑器。管理员和超级用户拥有更高权限；普通用户按 `permissions_json` 控制页面模块访问。企业身份源可通过 [LDAP/AD](../integration/ldap.md) 或 [OIDC](../integration/oidc.md) 接入。

## 审计日志

**平台运维 → 日志中心** 可按关键词、操作者、对象、动作、事件类型、成功/失败和时间范围查询，并按相同条件导出 JSON 或 CSV。

审计是现有中间件和业务接口的事件记录，不应宣称能自动覆盖任意未来代码路径。新增写接口时，贡献者需要补充审计事件和测试。日志中的敏感字段应经过脱敏；导出文件按敏感运维数据管理。

## API 安全

**系统管理 → API 安全** 管理机器调用使用的 API Key：

- 创建时选择 `ops`、`openapi` 等后端允许的 scope；
- 设置有效天数、每分钟限流和突发额度；
- 完整 Token 只在创建或轮换后显示一次；
- 可轮换或吊销，旧 Token 会立即失效。

示例：

```bash
curl -H "Authorization: Bearer ${BEACON_API_KEY}" \
  http://<server>:9991/open/ops/health
```

一个 Key 只用于一个系统和环境，按最小 scope 发放并定期轮换。不要把 Token 放进 URL、仓库、Issue 或浏览器截图。完整认证约定见 [认证鉴权](../api/authentication.md)。

## 其他入口

- **授权管理**：导入并诊断 Beacon 许可证；开源许可与产品运行许可证是两件不同的事。
- **开发入口**：查看当前实例暴露的 OpenAPI 与集成信息。
- **告警声管理**：维护布控任务可选择的提示音资源。
- **升级中心**：只接受符合部署约定的升级包；生产应用前应备份并验证回滚。

部署级安全、备份和密钥轮换见 [安全加固](../deploy/security-hardening.md)、[数据库与备份](../deploy/database-and-backup.md) 和 [密钥轮换](../deploy/secrets-and-rotation.md)。
