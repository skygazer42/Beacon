# 大屏监控

大屏页面把当前可播放的视频流分配到多个窗口，并在侧栏显示热点流和最近告警。它复用 MediaServer 播放页面，不单独实现一套播放器内核。

## 分屏与投放

当前支持 **1、2、4、9、16** 画面：

1. 用页面右上角的分屏选择器切换窗口数。
2. 点击目标窗口，使其成为当前窗口。
3. 点击“投放到 XX 号窗”，在流选择抽屉中搜索并选择在线流。
4. 使用“清空窗口”移除当前窗口分配。
5. 播放失败时使用窗口内重试；页面“刷新”会清空播放地址缓存并重新读取数据。

切换分屏时页面会按当前热点流和在线流重新整理窗口。分配状态是当前页面状态，不应视为服务端保存的长期布局。

## 页面信息

- **热点流概览**：来自后端返回的重点流及告警计数。
- **最近告警**：提供视频流或告警详情链接。
- **窗口信息**：显示展示名称、`app/name` 标识和告警数。
- **窗口分配**：可通过鼠标或键盘选择目标窗口。

当前页面没有已保存布局管理、拖拽交换、右键菜单、截图按钮、声音开关或自定义全屏工具栏。浏览器自身的 F11/全屏能力不属于 Beacon 页面功能。

## 为什么窗口为空

大屏只投放后端判定为可播放的流。数据库里存在一条记录并不足够：

1. 视频资源应处于启用状态；
2. MediaServer 应已成功建立代理，列表状态为“转发中”；
3. `app/name` 必须与在线媒体流一致；
4. Admin 能从 MediaServer 获取播放地址；
5. 浏览器能访问返回的 WebRTC/HLS/FLV 地址，反向代理和防火墙没有拦截相关端口。

排障时先在 **视频资源 → 播放 / WebRTC 自检** 验证单流，再回到大屏；这样可以区分源流、MediaServer、浏览器播放和窗口分配问题。
