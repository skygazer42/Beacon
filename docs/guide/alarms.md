# 报警管理

告警中心展示 Analyzer 或外部接口写入的告警，支持筛选、处置、媒体查看和证据导出。

## 列表与筛选

列表字段包括缩略图或视频入口、描述、工作流状态、视频流、算法、布控和创建时间。可组合使用：

- 状态：新告警、已确认、已解决、已忽略、已分配；
- 布控编号、视频流编号、算法编号；
- 日期范围。

常用组合可以保存为筛选预设；预设只保存筛选条件，不复制告警数据。页面还提供语义检索预览，只有后端已配置对应检索能力时才会返回结果。

## 批量处理

勾选告警后可执行：

- 标记已读；
- 批量已处理并填写备注；
- 恢复未处理；
- 删除。

删除会影响后续审计和证据导出，应按组织的数据留存制度授权使用。

## 告警详情

点击“详情”可查看图片、视频、基本字段、分配对象、处理人、备注和工作流时间线。详情动作包括：

- 分配给指定对象；
- 确认、解决或忽略；
- 按时间窗口执行跨镜头检索；
- 下载后端提供的媒体；
- 导出证据 ZIP、LabelMe 或 COCO。

LabelMe/COCO 导出要求告警仍有图片文件和检测框元数据。管理页面只从单条详情发起即时导出；批量数据集接口属于 OpenAPI 能力，不应误认为页面存在后台导出任务。

## 告警外发

当前保留的出口是 Webhook 和 Cloud：

- Webhook 使用配置的目标地址；设置签名密钥后会携带 `X-Beacon-Signature`。
- Cloud 用于边缘实例把告警上报给已接入的云端控制面。

配置与验签见 [Webhook 集成](../integration/webhook.md) 和 [云边控制面](../integration/cloud-remote-control-plane.md)。通知失败时先检查 Outbox、目标网络和签名配置，不要在日志中输出完整 Token 或请求体里的敏感媒体。

## 常见问题

- **只有记录没有图片/视频**：检查布控的截图数量、视频格式、上传目录权限和文件保留策略。
- **列表状态和批量“已处理”不一致**：系统同时兼容工作流状态和旧处理字段；刷新详情确认最终结果。
- **导出失败**：确认告警媒体仍在磁盘，运行账户可读，并查看接口返回的明确错误。
- **告警过多**：先调整布控报警间隔、阈值和 ROI，再评估是否需要更改留存策略。
