# 用户使用指南

欢迎使用 **Beacon 新一代 AI 视频分析系统**。本指南将帮助您全面了解和掌握系统的各项功能，从视频流接入到智能分析再到报警处理，为您提供完整的操作指导。

Beacon 管理界面默认运行在 **9991** 端口，请在浏览器中访问 `http://<服务器IP>:9991` 进入系统。

## 系统工作流程

Beacon 平台的核心工作流程如下图所示：

```
视频流接入 ──→ 创建布控任务 ──→ 实时智能分析 ──→ 报警生成与推送
    │                │                │                │
    ▼                ▼                ▼                ▼
 RTSP/RTMP      选择算法          检测目标事件      Webhook/Cloud
 FLV/HLS        绘制检测区域      按配置截图/录像   实时告警
 ONVIF发现      设置阈值与输出    生成结构化结果    报警存档
```

### 第一步：视频流接入

将摄像头或视频源的流地址添加到系统中。Beacon 支持以下协议：

| 协议 | 说明 | 典型地址格式 |
|------|------|-------------|
| RTSP | IP 摄像头最常用的协议 | `rtsp://user:pass@192.168.1.100:554/stream1` |
| RTMP | 实时消息传输协议 | `rtmp://192.168.1.100/live/stream1` |
| HTTP-FLV | HTTP 长连接 FLV 格式 | `http://192.168.1.100:8080/live/stream.flv` |
| HLS | HTTP 分片播放列表 | `http://192.168.1.100/live/stream.m3u8` |

### 第二步：创建布控任务

为视频流绑定分析算法，在画面上绘制矩形或多边形检测区域，并设置检测阈值和告警输出。

### 第三步：报警处理

当分析引擎检测到目标事件时，系统自动生成报警记录（含截图和录像片段），并通过已配置的通知通道推送。

### 第四步：回溯与处置

在报警管理页面查看历史报警、回放视频、更新处置状态并导出证据，辅助事后分析和决策。

## 其他管理功能

除上述核心功能外，Beacon 还提供以下系统管理能力：

- **系统设置**：配置平台全局参数，包括存储路径、报警保留策略、通知通道等
- **用户管理**：添加/编辑/删除系统用户，分配功能权限
- **审计日志**：记录所有用户操作，便于安全审计和问题追溯
- **API 密钥管理**：生成和管理 API 访问密钥，用于第三方系统集成

!!! tip "操作提示"
    系统左侧导航栏可快速切换各功能模块。页面右上角显示当前登录用户信息和系统通知。遇到操作问题时，可将鼠标悬停在界面元素上查看工具提示。
