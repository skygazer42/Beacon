# 算法管理

算法记录描述 Analyzer 如何加载模型、插件或外部推理 API。开源仓库不附带模型权重和专有推理 SDK；创建记录前必须自行取得可合法使用的运行时文件。

## 算法列表

列表展示编码、名称、类型、来源、状态和 Analyzer 引用数，并提供：

- 添加或编辑算法；
- 对单张图片执行一次测试推理；
- 查看版本、设置灰度版本或回滚；
- 删除未再使用的算法。

“引用数”来自当前 Analyzer 状态，不是性能评分。当前页面也没有自动基准测试或 FPS、显存、mAP 对比图。

## 创建算法

表单会根据算法类型显示不同字段：

- **基础算法**：本地模型或外部 API；追踪子类型只允许本地模型，语音子类型按当前契约使用 API。
- **行为 / 业务算法**：内置行为、外部 API 或动态库，并可声明直接 API 模式。
- **通用参数**：授权算法包、布控数量上限和模型并发数。

本地模型上传框接受 `.pt`、`.pth`、`.onnx`、`.xml`、`.engine`、`.trt`、`.weights`、`.enc` 等后缀。后缀被接受不代表当前机器一定具备对应后端：

- OpenVINO `.xml` 需要配套 `.bin`；
- YOLO `.weights` 需要配套 `.cfg`；
- TensorRT engine 通常与 GPU 架构、CUDA 和 TensorRT 版本绑定；
- `.pt/.pth` 是否可运行取决于构建中实际启用的加载器。

基础模型还可设置检测目标、输入宽高、模型精度标识、置信度阈值和 NMS 阈值。精度字段是配置元数据，系统不会仅凭选择 FP16/INT8 自动完成量化。

## 测试推理

算法列表和编辑页均提供单张图片的一次推理入口。选择实际部署设备后上传测试图片，返回结果用于验证“模型能否加载、接口能否调用、输出能否解析”。它不能代替视频场景验收或性能基准。

正式布控前建议：

1. 用若干有代表性的现场图片做一次推理；
2. 创建一条测试布控，在目标视频上观察一段时间；
3. 分别记录误报、漏报、处理延迟和资源占用；
4. 确认模型、标签和第三方 SDK 的许可证允许目标部署方式。

## 常见问题

- **GPU 后缀仍跑不起来**：后缀只是选择后端；检查驱动、CUDA/TensorRT/OpenVINO、模型兼容性和 Analyzer 构建选项。
- **编辑后任务没有变化**：已有布控可能仍持有旧实例，停止并重启相关任务。
- **测试图片成功但视频无结果**：检查布控目标类别、ROI、阈值、解码步进和实际流分辨率。
- **模型上传失败**：检查文件大小限制、配套文件和 Admin 上传目录权限。
