# 布控管理

布控任务把一路视频流和一个算法实例组合起来，并定义检测区域、阈值、告警素材与可选的推流叠加。

## 创建布控

进入 **布控中心 → 添加布控**：

1. 选择视频流。页面会同时读取数据库资源和 MediaServer 当前流列表。
2. 选择算法及 CPU、GPU、TensorRT、AUTO 或 NPU 变体。选项只生成运行时算法编码，是否真正可用取决于 Analyzer 的推理后端和模型。
3. 选择检测目标，并在预览画面上用矩形或多边形绘制 ROI。
4. 设置报警间隔、分类阈值、重叠阈值和解码步进。
5. 按需配置报警铃声、视频格式、截图数量、绘制模式和封面帧。
6. 需要输出分析流时打开推流，并配置文字或贴图 OSD。
7. 保存后返回列表，手动启动任务。

ROI 坐标按画面归一化保存。编辑前应先加载当前帧；源流不可用时仍可直接编辑 `polygon` 文本，但应在启动前验证区域与实际分辨率一致。

## 阈值说明

| 参数 | 作用 |
|---|---|
| `minInterval` | 同一任务两次告警之间的最小秒数 |
| `classThresh` | 分类或二阶段判定阈值；具体含义由算法决定 |
| `overlapThresh` | 目标与 ROI 的重叠判定阈值 |
| `decodeStride` | 每隔多少帧送入分析；增大会降低算力消耗，也可能漏掉短事件 |

不要把一套阈值复制到所有场景。应使用现场视频分别观察误报、漏报和端到端延迟，再逐项调整。

## 列表操作

列表支持搜索、启动、停止、编辑、快捷设置、复制和删除。快捷设置只修改解码抽帧、告警视频、截图和绘制模式，可选择保存后重启。

批量能力包括：

- 启动或停止已选任务；
- 选择一个源任务，复制到多个目标视频流；
- 打开布控日志并导出 JSON。

当前页面没有单独的“暂停”和时间计划按钮；需要停用时使用“停止”。

## 启动失败排查

1. 视频资源已启用，并且对应 `app/name` 在 MediaServer 在线。
2. 算法记录已启用，模型或插件文件存在，设备后缀与运行环境一致。
3. Analyzer 端口可达且授权允许该算法包和布控数量。
4. 查看 **布控日志** 和 Analyzer 日志中的首个错误，不要只根据列表中的最终状态判断。
5. 修改算法、ROI 或输出参数后，停止并重新启动任务验证。
