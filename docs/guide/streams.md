# 视频流管理

视频流是布控任务的数据源。当前管理页面负责保存流配置，并调用 MediaServer 建立或停止代理链路。

## 添加视频流

在 **视频资源 → 添加视频流** 中填写：

| 字段 | 说明 |
|---|---|
| 编号 | Beacon 内部唯一编号；创建后不应随意修改 |
| 分组（app）/ 名称（name） | MediaServer 中的流标识，通常组合为 `app/name` |
| 视频流地址 | 上游拉流地址 |
| 拉流类型 | RTSP、RTMP、HTTP-FLV、WS-FLV 或 HLS |
| 摄像头名称 | 页面展示名称 |
| 站点 / 楼层标签 | 可选筛选信息 |
| 备注 | 可选运维信息 |

页面没有“上传本地视频并自动建流”的能力。若要循环播放本地文件用于测试，请按 [端到端验收](../deploy/e2e-acceptance.md) 启动仓库自带的 RTSP 模拟器，或由 FFmpeg/MediaServer 提供可拉取地址。

## 状态与转发

列表中的两个状态含义不同：

- **启用 / 停用**：数据库中的资源状态，表示该视频流是否参与平台操作。
- **转发中 / 未转发**：MediaServer 是否已建立拉流代理。启用记录不会自动等于已经转发，除非打开“自动转发”或执行开启转发动作。

常用动作包括：

- 播放：进入单流播放器并请求当前可用播放地址；
- 启用 / 停用：修改资源状态；
- 开启 / 停止转发：建立或删除 MediaServer 代理；
- WebRTC 自检：查看浏览器播放链路诊断；
- 回讲：维护并启停该流的回讲配置；
- 云台：仅对已识别为 GB28181 的流显示；
- 转推代理：把当前流转推到指定目标；
- 编辑 / 删除：维护资源记录。

批量区支持选中流的转发启停、刷新转发状态、全部启动转发和 CSV 导入。CSV 列顺序为：

```text
昵称,视频流地址,备注,摄像头编号(可选)
```

## 搜索与设备发现

列表可按名称、编号或地址搜索，并在后端返回选项时按分组和站点筛选。ONVIF 发现是独立页面 **平台运维 → 设备扫描**；扫描到设备后再按页面提示导入视频流。

## 排查顺序

1. 确认源地址可从 MediaServer 所在主机访问；摄像头用户名和密码正确。
2. 启用资源后执行“开启转发”，再刷新转发状态。
3. 先看 MediaServer API/日志是否成功拉流，再检查播放器返回的 WebRTC、HLS 或 FLV 地址。
4. 分析任务仍无画面时，确认布控绑定的 `app/name` 与实际在线流一致。

摄像头凭据属于敏感信息。生产环境应保护数据库和备份，不要把带密码的 RTSP 地址粘贴到 Issue、日志或截图中。
